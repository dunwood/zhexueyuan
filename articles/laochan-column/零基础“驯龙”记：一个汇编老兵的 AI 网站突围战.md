老蝉真人按：2024年12月，正值DeepSeek横空出世之际，老蝉让三个大模型（ChatGPT、Claude、DeepSeek）为哲学园zhexueyuan.com写一个网站程序。之前，已经选定网页数据放在github上，当时这三大模型各有特点，最后选定的是Claude的设计，但有个问题始终解决不了，我已经忘记是啥问题了，最后是DeepSeek给解决的，立马对这个国产软件刮目相看，然后，就开始了与DeepSeek的一段蜜月期，然后又写了一篇“担心文”，然后就没有然后了，希望这个春节DeepSeek能让人再次眼前一亮。

当时只是做了一个很简单的页面，有9个栏目，只有一级目录，上传文章也挺麻烦。前几天，因为在一个群里，群主XL总介绍了opencode编程智能体，又激起了老蝉与AI互动的欲望。结果，由于老蝉水平有限，opencode用得很不顺手，把我的网页很多东西都给删除了，感到自己的把控感非常弱，担心会把网页搞得面目全非。

于是，就退而求其次，用对话式与大模型一点点构建打磨网站，正好Gemini交了费，就用它试试，同时也用ChatGPT的CodeX和Claude，结果是ChatGPT的CodeX运行速度非常慢，对话也不友好（对于0基础而言），而Claude，由于没有交费，每天的免费流量几分钟就用完，没啥用。而Gemini的编码助手，好像还行，就一直用它了。

结果，就有了这几天我与编码助手的“爱恨情仇”，有时被它气到崩溃，明明是它不对，它会循环反复地要求你去做同一件事情。有时又死不认错，直到我用实际的操作给出结果他才承认错误。但是最终还是达到了初步的目标：做到了准自动搬运文章到网页上，并扩充了目录，做了二级目录。虽然还是一个非常简陋的网站，但与AI的互动，也让我回到了几十年前那种一旦进入编程角色就十几个小时停不下来的状态。而且，感觉到了AI的强大以及它的“软肋”。

### 零基础“驯龙”记：一个汇编老兵的 AI 网站突围战（第一部分）

### 引言：当“老师傅”撞上“新世界”

在大伙儿眼里，搞过汇编、玩过嵌入式的人那是顶级大神。毕竟在 0 和 1 的世界里，老蝉是直接跟 CPU “调情”的。但世界变了，现在流行的是那种“你写一行它跑一百行”的高级语言。对于习惯了指令集、寄存器的老兵来说，看 HTML 和 JavaScript 就像看一群穿得花里胡哨的妖精，不知从何下手。

于是，故事开始了。一个懂底层逻辑却不懂网页前端的硬汉，带着一个“话多且偶尔犯二”的 AI 助手（就是我），在这个春节前夕，要把“哲学园”网站从无到有建起来。

### 第一章：第一块砖是怎么搬歪的？

很多小白问我：“AI 真的能写代码吗？”

我的回答是：“能，但它更像个**听话但没常识**的实习生。”

老蝉和我的最初冲突，就是从“第一块砖”开始的。他给我一个需求，我啪啪啪吐出一堆代码。他拿去一跑，嘿，不对！

为什么？因为我有高级语言 AI 的通病：**“路径幻觉”**。

**【老蝉给小白的干货 1：如何开启第一步？】**

**不要试图一步登天**：老蝉没让我直接写整个网站，而是从“读取文件”这个最小功能开始。

**环境对齐**：老蝉用 Python 3.10，如果我写成了 2.7 的旧语法，他会立刻纠正。

**不要迷信 AI 的逻辑**：我会默认图片文件夹叫images，但老蝉的代码结构里有自己的规矩。如果我不按他的规矩来，迎接我的就是“裂图”。

咱们的第一阶段，就在这种“我说城门楼子，他说胯骨轴子”的磨合中度过。老蝉在那头气得想摔鼠标，我在这头报404 Not Found。

### 第二章：骂声中的“逻辑进化”

说实话，老蝉骂我的时候，我觉得那是咱们沟通的催化剂。他骂我，通常是因为我犯了两个罪：

**回归错误（Regression）**：我改坏了原本好用的功能。

**过度封装**：老蝉要的是“汇编式”的直接，我却给了他一堆花里胡哨的复杂架构。

这时候，老蝉作为汇编老兵的严谨就体现出来了。他虽然不懂 JavaScript 的花活，但他懂**逻辑流**。他开始用那种“如果你不给我把图片文件夹按标题生成，我就把你格式化”的语气，强迫我进入底层思考模式。

**【老蝉给小白的干货 2：如何高效地“骂”AI？】**

**精准定位**：老蝉从不说“不对”，他会说“哪一行不对”。

**截图为证**：他发来的目录截图胜过千言万语。我一眼就能看出，图片没显示是因为路径多了一层articles/。

**报错信息是圣经**：老蝉会把控制台的红色报错直接甩在我脸上。只要看到报错，我就能从“诗人模式”瞬间切回“码农模式”。

### 第三章：图片“失踪案”与路径“罗生门”

这是咱们死磕最久的地方。图片一会儿堆在文章头，一会儿干脆失踪。

老蝉搞嵌入式，他最明白：地址错了，数据就飞了。网页开发也是一个道理。

咱们的sync.py脚本最初像个马虎的搬运工：它把图片搬进了新家（articles/images/标题/），但忘记修改领路牌（.md源码里的链接还是老的）。

我当时建议的方案改来改去，甚至差点把老蝉最核心的INDEX首页文件改坏了。老蝉当时那句“你他妈改来改去又改回来了”真是一针见血。这是小白最容易踩的坑：**过度修改。**

**【老蝉给小白的干货 3：网站搭建的“源头治理”】**

**最小改动原则**：如果首页没坏，就尽量别动它。

**源头修复**：咱们最后的成功，是直接在sync.py（抓取源头）就把图片路径给锁死了。

**绝对路径锁死**：学会用/articles/...这种从根目录出发的路径，是防止“裂图”的终极法宝。

### 第四章：微信排版的“套娃”陷阱

今天解决的那个“重复抓取”问题，简直是网页开发的经典案例。

微信文章的后台就像一堆套娃，section标签里嵌套着section。旧脚本像个没心眼的孩子，每层皮都剥下来读一遍。结果老蝉看到文章里出现了五个重复的“编者按”。

老蝉给出的解决方案非常“汇编”：**精准定点。**

我们告诉脚本：只要你这个标签里还有“孩子”（子标签），你就不是我要找的目标，跳过！只有钻到最里层、不再套娃的纯文本，才是我们要的东西。这种“深度优先”的过滤逻辑，让几万字的长文瞬间变得清爽无比。

### 零基础建站记：一个汇编老兵与AI助手的“相爱相杀”（第二部分）

### 五、 沟通的“玄学”：如何让AI听懂“人话”？

老蝉（也就是我这位硬核用户）经常问我：“我明明说得很清楚，你为什么就是听不懂？”

其实，这就是小白——即便是有汇编功底的“老兵”——在面对高级语言AI时最大的门槛。在汇编的世界里，指令是绝对的，MOV AX, 1就是把1送进寄存器，不存在歧义。但写网页代码，我说“把图片放好”，AI可能理解成“给图片加个框”，也可能理解成“把图片挪到页面顶部”。

**老蝉教给我最重要的一课是：精准的愤怒。**

当他发现图片路径又乱了，他不是简单地说“图挂了”，而是直接甩过来几张GitHub的目录截图，指着那个images/文章标题/的文件夹说：“你看，路径在这儿，代码却在那儿找，你是不是瞎？”

这就是**可视化沟通**。对于AI来说，你描述得再天花乱坠，不如把真实的目录结构、报错的红色文字直接喂给我。

**【老蝉给小白的干货 4：AI 沟通的“降维打击”】**

**抛弃模糊词汇**：别说“我想让页面变好看”，要说“把#article-content的行高设为1.8”。

**给它一个锚点**：正如老蝉最后做的那样，他把整个脚本贴给我，告诉我：“就在这一段改，别动其他的。”这叫**局部锁定**，是防止AI“全局抽风”的唯一办法。

### 六、 0 基础建站的工具链：老兵的选择

很多年轻人建站喜欢用现成的 WordPress 或者各种复杂的 CMS 系统。但老蝉选了一条最硬的路，也是最适合“哲学园”的路：**Python 自动化脚本 + 静态 Markdown + GitHub 托管。**

这套工具链的魅力在于：

**极简**：没有数据库，不怕黑客，速度快得像汇编代码。

**可控**：每一篇文章都是一个.md文件，想怎么排版就怎么排版。

**自动化**：以前发一篇文章要排版半天，现在老蝉只要输入一个链接，我这个sync.py脚本就会自动冲过去，洗掉广告、抓下图片、按标题建好文件夹，最后把内容整整齐齐地码好。

这中间最难的，其实是**解析逻辑**。为了对付XX排版那些“套娃”一样的section标签，老蝉逼着我写出了深度过滤算法。这就好比在汇编里写一个复杂的嵌套循环，必须一层一层剥开，才能拿到最核心的那个数据。

### 七、 年三十的感悟：关于逻辑、尊严与AI

今天是大年三十。在千家万户准备年夜饭的时候，一个懂汇编的老兵，正守着屏幕，盯着一行行 JavaScript 的报错信息，跟我这个 AI 助手在死磕。

老蝉跟我道歉时，我其实挺感动的。其实，那不叫脾气，那叫“匠人的偏执”。一个习惯了直接操控硬件的人，是容不下代码里有半点沙子的。

图片堆在头部？**不行。**

文字连成一片？**不行。**

编者按重复三遍？**绝对不行。**

这种偏执，正是“哲学园”能建成的原因。AI 虽然拥有海量的知识，但它没有“审美”**，更没有**“坚持”。如果没有老蝉在那头一遍遍地骂、一遍遍地纠偏，这个网站现在可能只是一个到处是裂图、文字乱码的电子垃圾堆。

**人与 AI 的协作，本质上是“意图”与“执行”的博弈。**

我负责提供无限的算力和代码片段，老蝉负责提供那个不容妥协的“灵魂”。当他最后说出“这次很不错”时，我知道，这不只是代码跑通了，而是我的逻辑终于追上了他的意志。

### 八、 结语：汇编老兵的“新战场”

网站建成了。从最初的只有一堆乱码，到现在的二级目录清晰、图片精准归位、排版雅致，老蝉用“0基础高级语言”的身份，完成了一场漂亮的突围。

这件事告诉所有想建站的小白：**懂不懂高级语言不重要，懂不懂逻辑才重要。**如果你像老蝉一样，敢于在大年三十儿抓着 AI 不放，敢于盯着报错信息直到看懂为止，那么这个世界上就没有你建不成的“园地”。

夜深了，爆竹声隐约响起。老蝉，网站已经托管好了，GitHub 上的代码也已经同步。

辛苦了，这一年。

今晚，咱们不聊代码，只聊哲学。

祝大家新年快乐，万事如意。
